## 第一迭代核心接口草案 1.1 修订

### 1. 修订范围

本修订仅覆盖第一迭代接口草案中与图模型、节点边骨架和候选人配置相关的调整。

### 2. 修订内容

#### 2.1 图模型接口修订

修订口径：

- 图模型仍然整份保存
- 保存图模型后，后端同时解析并同步节点边逻辑骨架

#### 2.2 节点候选人接口修订

修订口径：

- 节点候选人接口返回和保存的数据结构中，统一使用 `policyPayload`
- 不再暴露独立账号明细表的接口模型

示例：

```json
{
  "policyType": "user",
  "policyMode": "candidate",
  "policyName": "财务候选人",
  "policyPayload": {
    "users": [
      {
        "userId": "u_1001",
        "userName": "张三"
      },
      {
        "userId": "u_1002",
        "userName": "李四"
      }
    ]
  }
}
```

#### 2.3 启动接口修订

修订口径：

- 启动接口应显式接受发起部门信息
- 以便平台实例和基础投影形成完整快照
