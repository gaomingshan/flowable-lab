## 第一迭代节点与候选人配置能力清单 1.1 修订

### 1. 修订范围

本修订仅覆盖候选人模块的建模方式。

### 2. 修订内容

#### 2.1 候选人配置模型修订

修订口径：

- 第一迭代候选人仍然挂载在节点上
- 主配置对象保持结构化
- 动态策略明细不再拆独立账号明细表
- 统一通过 `policyPayload` 承载

#### 2.2 第一迭代账号模式示例

```json
{
  "policyType": "user",
  "policyMode": "candidate",
  "policyName": "财务候选人",
  "policyPayload": {
    "users": [
      {
        "userId": "u_1001",
        "userName": "张三"
      },
      {
        "userId": "u_1002",
        "userName": "李四"
      }
    ]
  }
}
```

#### 2.3 后续扩展方向

第二迭代扩展角色、部门、表达式时，继续复用同一个主配置表，只调整 `policyPayload` 的内部结构。
